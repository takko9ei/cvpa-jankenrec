# cvpa-jankenrec — 实时猜拳识别（纯传统 CV）

实时识别石头 / 剪刀 / 布（Rock / Scissors / Paper），**只用传统计算机视觉**，不依赖任何机器学习。
这是一个课设项目，核心目的是手写传统 CV 流水线。

---

## 硬约束（Hard Constraints）

1. **只允许 `OpenCV` + `NumPy`。** 不引入其它第三方库。
2. **禁止任何机器学习 / 深度学习手段**：不用 mediapipe、不用任何预训练模型、不用 dnn 模块、不用 haar/cascade 之类的现成检测器。
3. **禁止现成手势捷径**：特别地，**不允许使用 `cv2.convexityDefects`**（凸缺陷）这类直接给出手指信息的现成函数。手指计数必须自己用几何方法（如同心环采样）手写实现。
4. 一切几何 / 形态学 / 阈值逻辑都必须是我们自己写的传统 CV 步骤。

> 评判标准：能不能讲清楚每一步在做什么、为什么这么做。所以“自己实现”比“识别率高”更重要。

---

## 手势判定规则（Gesture Thresholds）

按“伸出的手指数”映射手势：

| 手指数 | 手势 | 英文 |
| ------ | ---- | ---- |
| 0–1    | 石头 | Rock |
| 2–3    | 剪刀 | Scissors |
| ≥4     | 布   | Paper |

胜负规则：石头 > 剪刀，剪刀 > 布，布 > 石头。

---

## 范围外（Out of Scope）

明确**不处理**以下情况，遇到直接放弃，不要为它们增加复杂度：

- 两只手相互接触 / 重叠（手与手粘连）。
- 复杂背景（默认假设背景与肤色差异明显、相对干净）。
- 手势旋转、侧视、手背朝向镜头等非正面手型。
- 多于两只手的混战场面（最多支持画面里同时出现两只分离的手用于判负）。

---

## 代码风格（Code Style）

- **脚本式、线性可读优先。** 不要软件设计模式（不要类、工厂、策略模式等）。函数就是一串顺序步骤。
- **所有可调参数集中在 `pipeline.py` 顶部的常量区**（大写命名）。改参数只改那一处，不在函数体里散落魔法数字。
- 代码内写**清晰的英文注释**，解释“这一步在做什么 / 为什么”。
- **每完成一个阶段，用中文逐行讲解关键部分**，方便课设答辩复述。

---

## 工作流纪律（Workflow Discipline — 必须严格遵守）

- **一次只实现当前指定的那一个阶段**（一个 pipeline 函数）。不要一口气把整条流水线写完。
- **严禁修改任何之前已经过关的函数、常量或文件**——包括 `skin_mask` 等已完成的部分。
- 如果认为某个已过关的部分需要改动，**必须先停下来明确说“我想改 X，因为 Y”，经用户同意后才能动**。
- **不得擅自“顺手优化”或“修复”已有代码。**
- 每个阶段先在 **`debug.py` 用静态图**（`data/` 里的样张）验证通过，再进入下一阶段。
- 验证 OK 后，才把该步接入 `main.py` 的实时循环。
- **不做任何 git 操作**（`git add` / `commit` / `push` 等一律由用户手动执行）；只允许编辑 `.gitignore` 这类文件本身。

---

## 文件结构（Files）

- `pipeline.py` — 核心流水线：顶部常量区 + 8 个处理函数。
- `debug.py`    — 单步调试：`show()` 并排显示中间结果，用静态图跑。
- `main.py`     — 实时主程序：开摄像头、读帧、显示、FPS、按 `q` 退出。
- `data/`       — 静态测试图（如 `paper.png`）。
- `venv/`       — 本地虚拟环境（OpenCV 4.13 + NumPy 2.5）。

## 流水线总览（Pipeline Overview）

```
frame
  └─ skin_mask        肤色分割 -> 二值掩膜
       └─ split_hands     连通域拆分 -> 每只手一张掩膜
            └─ palm_center    找掌心 + 内切圆半径
                 └─ crop_forearm  切掉手腕/前臂
                      └─ count_fingers  同心环采样数手指
                           └─ classify     手指数 -> 手势名
  judge   两个手势 -> 胜负
  draw    把结果画回原帧
```
